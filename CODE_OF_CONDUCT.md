# Code of Conduct

## Our Standard

This project expects contributors and maintainers to communicate professionally and constructively.

Examples of expected behavior:

- discussing technical disagreements in good faith
- giving actionable, specific feedback
- staying respectful in issues, pull requests, and review comments
- focusing criticism on ideas, code, and behavior rather than people

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- hostile or disruptive behavior
- deliberate bad-faith engagement
- publishing private information without permission

## Scope

This Code of Conduct applies to project spaces such as:

- GitHub issues
- pull requests
- code review discussions
- project documentation discussions

## Enforcement

Project maintainers may remove, edit, or reject contributions or discussions that violate these standards.
